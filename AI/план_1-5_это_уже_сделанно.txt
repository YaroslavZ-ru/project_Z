## Изменение 1: Создание структуры каталогов

### Что нужно сделать

Создать на диске следующую структуру папок (относительно корня проекта):
```
ai_terminator/          # корень проекта (имя может быть другим)
├── src/                # исходный код Python
│   └── __init__.py     # пустой файл, чтобы src был пакетом
├── data/               # базы данных, словари синонимов
├── models/             # загружаемые модели (fastText, transformers)
├── tests/              # тесты
├── logs/               # логи
├── configs/            # конфигурационные файлы
└── main.py             # точка входа (пока заглушка)
```

### Рекомендации

- Используй `pathlib.Path` для создания папок и файлов, это кроссплатформенно.
- Игнорируй ошибки, если папка уже существует (например, `exist_ok=True` в `Path.mkdir`).
- Файл `src/__init__.py` может быть пустым, но он необходим для того, чтобы Python распознавал `src` как пакет (возможны относительные импорты).
- `main.py` пока создай пустым или с минимальным содержанием (например, `print("Hello")`).

### Пример кода (иллюстрация)

```python
from pathlib import Path

PROJECT_ROOT = Path(__file__).parent  # если скрипт запускается из корня
DIRS = ["src", "data", "models", "tests", "logs", "configs"]
for d in DIRS:
    (PROJECT_ROOT / d).mkdir(exist_ok=True)

# Создать __init__.py в src
(PROJECT_ROOT / "src" / "__init__.py").touch(exist_ok=True)
# Создать main.py если его нет
(PROJECT_ROOT / "main.py").touch(exist_ok=True)
```

**Важно:** Этот код нужно будет выполнить один раз (можно сделать отдельным скриптом `setup.py`, а можно просто создать папки вручную, но для автоматизации лучше написать скрипт). В дальнейшем структура предполагается существующей.
