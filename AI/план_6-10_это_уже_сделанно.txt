# План реализации AI-Terminator (Часть 2 из 8) — ИСПРАВЛЕННАЯ ВЕРСИЯ (изменения 6–10)

## Общие принципы для исправленной версии

- Все модули должны получать пути к файлам (словарь синонимов) и настройки из конфигурации (класс `Config`, созданный в изменении 5).
- Соблюдаем точное соответствие формуле из ТЗ:  
  `V = 0.7 * V_term + sum(0.3/N * V_hint_i) + sum(0.1 * V_syn_j)`  
  где `V_syn_j` — вектор каждого синонима, а **общая сумма весов всех синонимов равна 0.1**.
- Синонимы выбираются не более 2 на каждый исходный токен.
- Лемматизатор обрабатывает составные термины (разбивает на слова).
- Добавлены проверки максимальной длины `term` (100 символов) и каждого `hint` (50 символов) после очистки.
- Очистка текста корректно удаляет краевые дефисы, сохраняет буквы (русские и латинские), цифры, пробелы и внутренние дефисы.
- Логирование ошибок и предупреждений (используем стандартный `logging` с уровнем из конфига).
- Кэширование результатов лемматизации для производительности.

## Изменение 6: Лемматизатор (синглтон) с поддержкой составных терминов и кэшированием

### Что нужно сделать

Создать модуль `src/lemmatizer.py` с классом `Lemmatizer`, который:
- Инициализирует `pymorphy2.MorphAnalyzer()` один раз (синглтон).
- Предоставляет метод `lemmatize_word(word: str) -> str` для одного слова (с кэшированием).
- Предоставляет метод `lemmatize(phrase: str) -> List[str]`, который разбивает фразу на слова, лемматизирует каждое через `lemmatize_word` и возвращает список лемм.
- При выборе варианта разбора из нескольких, выбирает вариант с максимальным `score` (поле `score` у `ParsedWord`).
- В случае ошибки (например, пустая строка) возвращает исходную строку для отдельного слова, а для фразы возвращает пустой список (или исходную строку без изменений).

### Рекомендации по реализации

- Кэш можно реализовать как `dict` внутри класса: `self._cache: Dict[str, str]`.
- Для получения лучшего варианта: `best = max(parsed, key=lambda p: p.score)`.
- Если слово невозможно распарсить (например, число), вернуть его в нижнем регистре.
- Для метода `lemmatize_phrase`: если фраза пуста, вернуть пустой список. Иначе разбить по пробелам, обработать каждое слово, отфильтровать пустые результаты.

### Пример (иллюстрация)

```python
import pymorphy2
from typing import List, Dict

class Lemmatizer:
    _instance = None
    _cache: Dict[str, str] = {}

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance._morph = pymorphy2.MorphAnalyzer()
        return cls._instance

    def lemmatize_word(self, word: str) -> str:
        if not word:
            return ""
        if word in self._cache:
            return self._cache[word]
        try:
            parsed = self._morph.parse(word)
            if parsed:
                best = max(parsed, key=lambda p: p.score)
                res = best.normal_form
            else:
                res = word.lower()
        except Exception:
            res = word.lower()
        self._cache[word] = res
        return res

    def lemmatize_phrase(self, phrase: str) -> List[str]:
        if not phrase:
            return []
        words = phrase.split()
        return [self.lemmatize_word(w) for w in words if w]
```

### Проверка

`Lemmatizer().lemmatize_phrase("ключи гаечные")` → `["ключ", "гаечный"]`.

## Изменение 7: Словарь синонимов с ограничением количества и правильной загрузкой

### Что нужно сделать

Создать файл `data/synonyms.json` в формате: ключ — лемма термина или подсказки, значение — **список синонимов (лемм)** (без весов, так как вес фиксирован 0.4?).  
Но согласно ТЗ: «Синонимам присваивается понижающий вес 0.4». Однако **формула векторизации** даёт суммарный вес всех синонимов = 0.1. Следовательно, индивидуальный вес каждого синонима вычисляется как `0.1 / (общее количество всех синонимов)`. Это более разумно, чем использовать 0.4. Чтобы не противоречить ТЗ, нужно придерживаться формулы. Поэтому словарь синонимов не должен хранить веса; веса будут назначены алгоритмически.

Таким образом, формат `synonyms.json`:

```json
{
    "ключ": ["инструмент", "отмычка"],
    "техника": ["механизм"],
    "вращение": ["поворот"]
}
```

Класс `SynonymDict` в `src/synonyms.py`:
- Загружает JSON при инициализации (путь из конфига).
- Предоставляет метод `get_synonyms(lemma: str, max_synonyms: int = 2) -> List[str]`, возвращающий **не более max_synonyms** синонимов (первые из списка).
- Если лемма отсутствует, возвращает пустой список.
- Обрабатывает ошибки: если файл не найден, выводит предупреждение через `logging.warning` и создаёт пустой словарь.

### Рекомендации

- В будущем можно расширить выбор синонимов по релевантности (например, по частоте), но пока берём первые N.
- Использовать `logging.getLogger(__name__)`.

### Пример

```python
import json
import logging
from typing import List

logger = logging.getLogger(__name__)

class SynonymDict:
    def __init__(self, json_path: str):
        try:
            with open(json_path, 'r', encoding='utf-8') as f:
                self._data = json.load(f)
        except FileNotFoundError:
            logger.warning(f"Файл синонимов не найден: {json_path}. Будет использован пустой словарь.")
            self._data = {}
        except json.JSONDecodeError as e:
            logger.error(f"Ошибка парсинга JSON: {e}")
            self._data = {}

    def get_synonyms(self, lemma: str, max_synonyms: int = 2) -> List[str]:
        return self._data.get(lemma, [])[:max_synonyms]
```

## Изменение 8: Очистка текста (исправленная)

### Что нужно сделать

Функция `clean_text(text: str) -> str` в `src/text_cleaner.py` должна:
- Удалять все символы, кроме:
  - букв (русские `а-яё`, латинские `a-z`)
  - цифр `0-9`
  - пробелов
  - дефиса `-` (но не в начале и не в конце слова)
- Приводить к нижнему регистру.
- Удалять лишние пробелы.
- Удалять начальные и конечные дефисы, а также дефисы, которые остались после удаления символов.
- Возвращать пустую строку, если результат пуст.

**Алгоритм:**
1. Заменить любые символы, не входящие в разрешённый набор, на пробел (чтобы не слипались слова).
2. Привести к нижнему регистру.
3. Заменить последовательности пробелов на один пробел.
4. Удалить пробелы и дефисы по краям.
5. Удалить дефисы, которые остались внутри (например, если было `-a-` → после удаления краевых станет `a`? Лучше оставить внутренние дефисы.

Более простой и надёжный способ: использовать регулярное выражение, которое заменяет все небуквенно-цифровые символы (кроме пробелов и дефисов) на пробелы, затем нормализовать.

```python
import re

def clean_text(text: str) -> str:
    # Заменяем нежелательные символы на пробел
    cleaned = re.sub(r'[^а-яёa-z0-9\s-]', ' ', text, flags=re.IGNORECASE)
    # Приводим к нижнему регистру
    cleaned = cleaned.lower()
    # Убираем лишние пробелы
    cleaned = re.sub(r'\s+', ' ', cleaned).strip()
    # Удаляем дефисы по краям
    cleaned = cleaned.strip('-')
    return cleaned
```

### Проверка

`clean_text("  -Ключ-гаечный! (размер 12)  ")` → `"ключ-гаечный размер 12"`.

## Изменение 9: Предобработка (базовая, с проверками)

### Что нужно сделать

В `src/preprocess.py` реализовать функцию `preprocess(term: str, hints: List[str], config: Config) -> dict`. Она должна:

1. Применить `clean_text` к `term` и каждому `hint`. Отфильтровать пустые подсказки.
2. Проверить, что после очистки `term` не пуст.
3. Проверить максимальную длину: `len(clean_term) <= 100` и `all(len(h) <= 50 for h in clean_hints)`. При нарушении вернуть `{"status": "error", "message": "..."}`.
4. Удалить дубликаты подсказок (сохраняя порядок: `dict.fromkeys()`).
5. Лемматизировать термин (через `Lemmatizer().lemmatize_phrase()`) и каждую подсказку (отдельно).
6. Вернуть словарь, содержащий:
   - `status: "ok"`,
   - `original_term`, `original_hints`
   - `clean_term`, `clean_hints`
   - `term_lemmas` (список лемм терма),
   - `hints_lemmas` (список списков лемм для каждой подсказки),
   - `all_lemmas` (список всех лемм),
   - **пока без весов**.

### Рекомендации

- Для удаления дубликатов: `cleaned_hints = list(dict.fromkeys(cleaned_hints))`.
- Логировать, если были отброшены дубликаты или пустые подсказки (уровень INFO).

### Пример возвращаемого словаря


## Изменение 10: Расширение предобработки синонимами с правильными весами согласно ТЗ

### Что нужно сделать

Модифицировать `preprocess`, чтобы она принимала `synonym_dict: SynonymDict` и возвращала **список токенов с весами**, строго по формуле:

- Вес исходных токенов терма (каждого слова) = `0.7 / len(term_words)`.
- Вес исходных токенов подсказок (каждого слова из всех подсказок) = `0.3 / total_hint_words`, где `total_hint_words` — общее количество слов во всех подсказках (после очистки и лемматизации).
- Для каждого исходного токена терма и подсказок добавляются **синонимы** (не более 2, полученные через `synonym_dict.get_synonyms(lemma, max_synonyms=2)`). **Суммарный вес всех синонимов (всех вместе) должен быть 0.1**, равномерно распределённый между всеми синонимами.

**Алгоритм вычисления весов синонимов:**
1. Собрать все синонимы из всех токенов (терм + подсказки) в один список, сохраняя привязку к исходному токену (для возможного учёта, но вес одинаков для всех).
2. Если общее количество синонимов `M > 0`, то вес каждого синонима = `0.1 / M`.
3. Если `M = 0`, синонимы не добавляются.

**Важно:** Мы игнорируем «понижающий вес 0.4» из текста ТЗ, потому что он противоречит формуле векторизации. Формула приоритетна. Для ясности можно добавить комментарий, что 0.4 относится к логике выбора синонимов, но не к итоговым весам.

**Структура `tokens_with_weights`:** список кортежей `(token: str, weight: float)`. Токены — это **леммы** исходных слов и леммы синонимов. Синонимы тоже должны быть лемматизированы (предполагается, что словарь уже содержит леммы).

### Детальные шаги

1. Получить `term_lemmas` и `hints_lemmas` из предыдущего шага.
2. Вычислить `term_weight_per_word = 0.7 / len(term_lemmas)` (если `term_lemmas` пуст — ошибка).
3. Создать список `tokens_weights = []`.
4. Добавить каждый токен терма: `(lemma, term_weight_per_word)`.
5. Собрать все синонимы для токенов терма в отдельный список `all_synonyms = []`. Для каждого `lemma` из `term_lemmas` получить синонимы через `synonym_dict.get_synonyms(lemma)` (не более 2) и добавить в `all_synonyms` (каждый синоним может дублироваться — позже сгруппируем).
6. Аналогично для подсказок: вычислить `hint_total_words = sum(len(lst) for lst in hints_lemmas)`. Если `hint_total_words > 0`, то `hint_weight_per_word = 0.3 / hint_total_words`. Для каждого слова из `hints_lemmas` добавить `(lemma, hint_weight_per_word)` и собрать синонимы в `all_synonyms`.
7. **Удалить дубликаты синонимов** (оставить уникальные леммы). Если синоним совпадает с исходным токеном (редко), его можно исключить или оставить — особой роли не играет.
8. Вычислить `synonym_weight = 0.1 / len(all_synonyms)` если список не пуст.
9. Добавить каждую уникальную лемму синонима в `tokens_weights` с весом `synonym_weight`.
10. Итоговый список `tokens_with_weights` может содержать дубликаты (например, один и тот же токен появился и как синоним, и как исходный). Это допустимо, при векторизации веса суммируются, что соответствует усреднению.

### Пример расчёта для `term="ключ"` (1 слово), `hints=["техника"]` (1 слово), синонимы: "ключ" → ["инструмент", "отмычка"], "техника" → ["механизм"].
- `term_lemmas = ["ключ"]`, `term_weight = 0.7/1 = 0.7`. Добавляем `("ключ", 0.7)`.
- `hint_weight = 0.3/1 = 0.3`. Добавляем `("техника", 0.3)`.
- Синонимы: "инструмент", "отмычка", "механизм" → M=3, вес каждого = 0.1/3 ≈ 0.0333.
- Добавляем `("инструмент", 0.0333)`, `("отмычка", 0.0333)`, `("механизм", 0.0333)`.

Итоговые веса: суммарный вес исходных токенов = 1.0, синонимов = 0.1.

### Обработка крайних случаев

- Если `term_lemmas` пуст (термин после очистки и лемматизации не дал слов) — вернуть ошибку.
- Если `hint_total_words = 0`, вес подсказок не добавляется.
- Если нет синонимов, просто не добавляем их.

### Реализация в коде (иллюстрация)

```python
from typing import List, Tuple
from src.synonyms import SynonymDict
from src.lemmatizer import Lemmatizer

def preprocess(term: str, hints: List[str], config, synonym_dict: SynonymDict) -> dict:
    # ... (очистка, проверки, лемматизация) ...
    term_lemmas = lemmatizer.lemmatize_phrase(clean_term)
    if not term_lemmas:
        return {"status": "error", "message": "Термин не содержит значимых слов после лемматизации"}

    hints_lemmas = [lemmatizer.lemmatize_phrase(h) for h in clean_hints if h]
    # ... удаление дубликатов hints_lemmas?...

    tokens_weights = []
    # Терм
    term_weight_per_word = 0.7 / len(term_lemmas)
    for lemma in term_lemmas:
        tokens_weights.append((lemma, term_weight_per_word))

    # Подсказки
    hint_total_words = sum(len(lst) for lst in hints_lemmas)
    all_synonyms = set()
    if hint_total_words > 0:
        hint_weight_per_word = 0.3 / hint_total_words
        for hint_list in hints_lemmas:
            for lemma in hint_list:
                tokens_weights.append((lemma, hint_weight_per_word))
                # Сбор синонимов
                for syn in synonym_dict.get_synonyms(lemma, max_synonyms=2):
                    all_synonyms.add(syn)
    else:
        # Если нет подсказок, всё равно собираем синонимы с терма
        for lemma in term_lemmas:
            for syn in synonym_dict.get_synonyms(lemma, max_synonyms=2):
                all_synonyms.add(syn)

    # Добавляем синонимы
    if all_synonyms:
        syn_weight = 0.1 / len(all_synonyms)
        for syn in all_synonyms:
            tokens_weights.append((syn, syn_weight))

    # Возвращаем результат
    return {
        "status": "ok",
        "original_term": term,
        "original_hints": hints,
        "clean_term": clean_term,
        "clean_hints": clean_hints,
        "term_lemmas": term_lemmas,
        "hints_lemmas": hints_lemmas,
        "tokens_with_weights": tokens_weights
    }
```

## Дополнительные замечания

- Все модули должны импортировать конфигурацию, но сама `preprocess` получает `config` только для доступа к параметрам (например, `max_synonyms` можно вынести в конфиг). Пока максимальное количество синонимов (2) можно захардкодить.
- Логирование: добавлять `logger.info` при успешной предобработке, `logger.warning` при отсутствии синонимов или дубликатах.
- Тестирование: после реализации всех изменений 6–10 можно написать небольшой скрипт `test_preprocess.py`, который загружает конфиг, инициализирует `SynonymDict`, вызывает `preprocess` для разных вариантов и проверяет веса.

После выполнения этих изменений шаг 1 конвейера будет полностью соответствовать ТЗ и готов к интеграции с шагом 2 (векторизация).
